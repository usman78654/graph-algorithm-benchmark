====================================
ANALYSIS
====================================

1. BFS vs DFS:
   - BFS: 0.056346 sec (FASTER)
   - DFS: 0.071730 sec
   - Difference: DFS is 1.27x slower (due to Python list operations)
   - Using deque for BFS provides O(1) popleft vs O(n) for list.pop(0)

2. Shortest Paths:
   - Dijkstra (0.129538 sec) OUTPERFORMS Bellman-Ford (0.487246 sec)
   - Speedup: 3.76x faster
   - Dijkstra uses min-heap priority queue (optimal)
   - Without min-heap, both would be much slower

3. MST Algorithms:
   - Kruskal (0.419826 sec) OUTPERFORMS Prim (0.798253 sec)
   - Speedup: 1.90x faster
   - Kruskal: Edge sorting + Union-Find (very efficient)
   - Prim: Priority queue heap operations
   - Both produced valid MSTs with different edge weights

4. Diameter Calculation:
   - 47.057217 seconds for 500 nodes
   - Extrapolated for all 35,776 nodes would be ~3400 seconds
   - Reason: Floyd-Warshall or all-pairs shortest path is O(V³)
   - Practical limitation: diameter on large graphs requires optimization

5. Data Structure Impact:
   - Deque vs List: Critical for BFS performance
   - Heap vs Array: Essential for Dijkstra/Prim efficiency
   - Union-Find: Enables Kruskal's fast implementation
   - Memoization: Could optimize diameter calculations
